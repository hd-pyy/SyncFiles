# Chinese Language Mode Design

## Goal

Add a point-and-click language selector to the SyncFiles desktop app so the user can use the app in Chinese, while keeping English available.

## Scope

The first version supports two UI languages:

- Chinese, shown as `中文`.
- English, shown as `English`.

Chinese is the default language at startup. The user can switch languages from the main window without restarting the app.

## User Experience

The main window adds a language selector near the top of the app. Selecting `中文` updates the visible UI text to Chinese. Selecting `English` updates the visible UI text to English.

The following user-facing text is translated:

- Window titles.
- Labels.
- Buttons.
- Tab names.
- Dialog titles and messages.
- Log messages generated by the app.
- Conflict action names shown in the conflict list and action dialog.

File paths, file names, Android paths such as `/sdcard`, and ADB command output are not translated.

## Architecture

Translations live in a small dedicated module instead of being scattered through the UI code. The UI asks this module for text by key and current language.

The Tkinter app keeps references to widgets whose text changes when the selected language changes. On language change, the app refreshes those labels, buttons, tabs, window titles, and existing conflict-list action labels.

## Testing

Automated tests cover:

- Chinese is the default language.
- Known text keys translate in both Chinese and English.
- Conflict action labels translate correctly.
- Building copy operations from a sync plan remains unchanged by the language work.

The full test suite remains the main verification command.
