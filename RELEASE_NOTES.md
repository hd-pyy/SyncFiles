# SyncFiles v0.2.0

本版本重点优化同步主界面和传输稳定性。UI 改为左右双栏文件面板,中间提供明确的方向同步按钮,并新增 SFTP 同步能力。

## 下载

- [SyncFiles-0.2.0-win64.exe](https://github.com/hd-pyy/SyncFiles/releases/download/v0.2.0/SyncFiles-0.2.0-win64.exe) (~22 MB,单文件,内含 Python 3.13 + Tcl/Tk + adb 及所需依赖)
- 适用于 **Windows 10 / 11 64 位**,双击即可运行,**无需**安装 Python 或 platform-tools

## 本版本更新

- **双栏文件视图**:左右两栏分别展示所选硬盘/手机/SFTP 文件夹的文件,包含名称、大小、修改时间和差异状态。
- **方向同步按钮**:中间按钮明确区分同步到右侧 / 同步到左侧,支持只执行指定方向的同步。
- **SFTP 模式**:支持硬盘与 SFTP 远程文件夹之间扫描、预览、冲突处理和同步。
- **手机模式校验**:浏览、扫描和同步手机文件夹前会检查设备状态,避免未连接或未授权时误操作。
- **ADB 稳定性**:保留 adb stdout/stderr 详情,增加命令日志,并对瞬时多设备/幽灵设备错误进行短重试。
- **修改时间显示**:差异列表中使用可读的本地时间格式展示文件修改时间。

## 已知限制

- **Windows SmartScreen 警告**:本 exe 未签名,首次运行可能被 SmartScreen 拦截。请点击"**更多信息**"→"**仍要运行**"继续。
- **仅支持 64 位 Windows**:未在 32 位 Windows 上验证。
- **未自动更新**:本版本不包含自动更新机制,后续版本需手动下载。
- **不会传播删除**:任一侧被删除的文件不会在另一侧删除,这是当前同步策略的有意行为。

# SyncFiles v0.1.0

首个公开版本。Windows 优先的桌面文件同步工具,支持硬盘↔硬盘与硬盘↔Android 手机(通过 ADB)两种模式,执行**双向补全式同步**:缺失文件从对端补齐,大小/修改时间不一致的文件标记为冲突,不会传播删除。

## 下载

- [SyncFiles-0.1.0-win64.exe](https://github.com/hd-pyy/SyncFiles/releases/download/v0.1.0/SyncFiles-0.1.0-win64.exe) (~15 MB,单文件,内含 Python 3.13 + Tcl/Tk + adb 及 6 个 DLL)
- 适用于 **Windows 10 / 11 64 位**,双击即可运行,**无需**安装 Python 或 platform-tools

## 本版本包含的功能

- **两种同步模式**
  - 硬盘模式:两个本地文件夹之间的双向补全同步
  - 手机模式:本地文件夹与 Android 手机(`/sdcard` 下的任意路径)之间的双向补全同步,自动调用内置 adb
- **冲突处理**:同名文件大小或修改时间不一致时标记为冲突,UI 中双击可选择保留方式(本地版 / 对端版 / 跳过)
- **预览再执行**:先扫描生成差异列表,确认后再开始同步,避免误操作
- **取消支持**:扫描与同步过程支持中途取消
- **路径校验**:扫描与同步前校验文件夹名,避免非法路径触发错误
- **中英双语**:默认中文,可在主窗口顶栏切换 `中文` / `English`
- **adb 解析顺序**:`SYNCFILES_ADB` 环境变量 → `PATH` 中的 adb → exe 同目录的 adb.exe → 内置 fallback

## 已知限制

- **Windows SmartScreen 警告**:本 exe 未签名,首次运行会被 SmartScreen 拦截。请点击"**更多信息**"→"**仍要运行**"继续。如需消除此提示,需购买代码签名证书。
- **仅支持 64 位 Windows**:未在 32 位 Windows 上验证,32 位系统无法运行此 exe。
- **未自动更新**:本版本不包含自动更新机制,后续版本需手动下载。
- **不会传播删除**:任一侧被删除的文件不会在另一侧删除,这是当前同步策略的有意行为,不是 bug。
- **手机模式依赖 USB 调试**:需在手机开发者选项中开启 USB 调试,并在首次连接时授权本机。
